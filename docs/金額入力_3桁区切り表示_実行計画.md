# 金額入力_3桁区切り表示_実行計画

## 目的
利用者詳細画面・まとめて入力画面の「入金登録」「出金登録」フォームにおいて、金額入力欄の表示だけを `3桁区切り`（例: `10000` → `10,000`）にする。

## 現状把握（送信値の前提）
- `app/residents/[id]/page.tsx`
  - 入金/出金モーダルの金額入力は `type="number"`、`value={formData.amount}`
  - バリデーション/送信で `parseFloat(formData.amount)` を前提にしている
- `app/facilities/[id]/bulk-input/page.tsx`
  - 入金/出金モーダルの金額入力は `type="number"`、`value={formData.amount}`
  - バリデーション/送信で `parseFloat(formData.amount)` を前提にしている

重要: `parseFloat("10,000")` は `10` になってしまうため、`formData.amount`（=DB/送信に使う値）にはカンマ付き文字列を入れない。

## 実現方針（表示だけ変える）
`formData.amount` は従来どおり「数値の生文字列（カンマなし）」として保持し、入力欄の見た目だけを `Intl.NumberFormat` で `3桁区切り` 表示に変える。

## 実装アプローチ（2案）

### 案A（推奨）: 入力欄の種類を `text` にして、表示は整形、内部値はサニタイズ
- 金額入力を `type="text"` に変更
- 内部状態 `formData.amount` は「数字のみ（0-9）」に正規化（例: `e.target.value` からカンマと非数字を除去）
- `value`（表示）には `formatAmount(formData.amount)`（例: `10,000`）を返す

注意点:
- 入力中に毎回整形すると、入力カーソル位置が少し不自然になる可能性がある（要動作確認）

### 案A-variant（許容しやすい）: フォーカス中は生入力、フォーカス外で整形
- フォーカス中: `value` は `formData.amount`（カンマなし）
- `onBlur` で整形された表示（カンマあり）に切替

この案は案Aよりカーソル問題が起きにくい。

### 案B: `type="number"` は維持し、別要素に整形表示を併記
- 入力欄自体は従来維持（=カーソル問題が起きにくい）
- ただし「入力欄の中に表示」という要件からは外れる可能性がある

## 実行タスク（変更範囲）
1. `app/residents/[id]/page.tsx`
   - 入金/出金モーダル内の金額入力（`formData.amount`）
   - 表示整形用の関数追加（または共通化）
   - `onChange` のサニタイズ（カンマ除去・非数字除去）追加
   - （必要なら）`onBlur` の整形切替
2. `app/facilities/[id]/bulk-input/page.tsx`
   - 入金/出金モーダル内の金額入力（`formData.amount`）
   - 上と同様の整形/サニタイズを適用
3.（任意）共通化
   - `lib/` に `formatAmount` / `sanitizeAmountInput` を追加し、両画面で使い回す

## 成功条件（動作確認観点）
- DB/送信値は従来通り整数として `parseFloat` される（カンマ付き文字列が `formData.amount` に入らない）
- 金額入力欄の表示のみが `3桁区切り` になる
- 既存のバリデーション（`1円以上の整数`）が壊れない
- 「次の入力」「編集（カード再投入）」でも同様に表示が正しく行われる

## 未確定事項（ユーザー確認したいこと）
- 入力中も即座に `3桁区切り`（例: `10000` → 途中で `10,000`）にしたいですか？
  - 1. はい（案A寄り）
  - 2. フォーカス外（確定後）だけ整形で良い（案A-variant）
