# 利用者入力画面：先月より繰越を明細先頭に表示する — 可否・実装計画

## 結論：可否

**実現可能です。**

理由：

- 繰越金額は **DBの既存取引から計算できる** 値です。`lib/balance.ts` の `calculateBalanceUpToMonth` と、PDF 用の `pdf/utils/transform.ts` で使っている「前月末までの残高」と **同じルール**（`in`/`out`/`past_correct_*` を加減算、`correct_in`/`correct_out` は除外）を流用できます。
- 画面は現在、**API が返した配列をそのまま** テーブル表示しているだけなので、**先頭に仮想行を1行足す**（または API で先頭に合成オブジェクトを載せる）設計にできます。
- PDF 側では既に `_isCarryOver` による **「前月より繰越」行の合成** が実装済みであり、画面でも同様の考え方を適用できます。

## 現状整理

| 箇所 | 内容 |
|------|------|
| `app/api/residents/[id]/route.ts` | 当月の取引のみ `calculateBalance` 後にフィルタして返却。**繰越行はない**。 |
| `app/residents/[id]/page.tsx` | `transactions` を `map` で明細表示。 |
| `pdf/utils/transform.ts` | 前月末残高を計算し、非0なら **「前月より繰越」** 行を追加（印刷用）。 |

用語：**「先月より繰越」** と **「前月より繰越」** — 実装時は PDF・帳票と表記を揃えるか、画面だけ「先月」にするかを決める（仕様確認事項）。

## スコープの前提（要確認）

- **対象画面**は本ドキュメントでは **利用者ごとの入力画面** `/residents/[id]` を主対象とする。
- **一括入力** `/facilities/[id]/bulk-input` は、明細が **施設内の全利用者の取引を日付順に混在** 表示するため、「表の一番上に1行だけ繰越」では意味が通らない。**利用者ごとに、その利用者の当月最初の取引の直前に繰越行を差し込む** など別仕様が必要。本要件が一括入力にも及ぶかは別途確認。

## 実装方針（案）

### 1. 繰越金額の定義

- 表示中の `year` / `month` に対し、**その月の1日より前の時点での累積残高**（＝前月末残高）。
- 計算：`calculateBalanceUpToMonth(transactions, yPrev, mPrev)`  
  - `mPrev`：`month === 1` のときは `year - 1, 12`、それ以外は `year, month - 1`。

### 2. 行を出す条件

- **PDF と同様「非0のときだけ行を出す」** か、**0円も常に1行出す** かはビジネス判断（実装前に決定）。

### 3. データの持ち方（どちらか）

| 案 | メリット | デメリット |
|----|----------|------------|
| **A. API で合成**（`route.ts` で配列先頭に追加） | 画面・将来の他クライアントで一貫 | レスポンス型に「仮想行」フラグが必要 |
| **B. フロントで合成**（fetch 後に `useMemo` で先頭へ） | API を触らずに試せる | 他画面で同じデータを使うと重複実装の恐れ |

推奨：**A（API）** — 残高表示と繰越の整合をサーバで一本化しやすい。

### 4. 仮想行の表現

- `id`：DBと衝突しない値（例：負の定数、または `null` + `key` は文字列 `"carryover-${year}-${month}"`）。
- `transactionDate`：**前月末日**（PDF と揃えると、日付ソートの意味づけと説明がしやすい）。
- `description`：**「先月より繰越」または「前月より繰越」**（表記統一）。
- `transactionType`：表示用に **`in` / `out` のどちらか**、または **`carryover` 等を追加** して区分バッジ・金額の正負を専用表示（後者の方が「訂正」ボタン誤表示を防ぎやすい）。
- **操作列**：編集・削除・訂正・コピーは **出さない**（読み取り専用行）。

### 5. 残高列との整合

- 現在の利用者APIは、当月フィルタ後の各行に付く `balance` は **`calculateBalance` による通算残高**（当月以前の取引をすべて含む累積）。
- 繰越行の **残高**は **前月末までの累積**＝その月の取引がまだ無い時点の残高と一致させる。
- 既存の各行の `balance` は変更不要（繰越行はその「直前」のスナップショットとして表示する）。

### 6. テスト観点（実装時）

- 前月末残高が正負ゼロ以外のとき、先頭行が繰越になること。
- `correct_in` / `correct_out` が繰越計算に含まれないこと（既存ロジックと一致）。
- 1月を選んだとき **前年12月末** までが繰越になること。
- 新規利用者で前月まで取引がない場合は繰越0（行を出さない場合は明細だけの表示）。
