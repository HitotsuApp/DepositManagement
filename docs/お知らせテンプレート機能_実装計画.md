# 預り金明細書【お知らせ】テンプレート機能 実装計画

> **実装済み**（2026-03-12）

## 1. 実装可否の判断

**結論: 実装可能です。**

現行の仕組みを確認した結果、以下の理由で実装可能と判断しました。

- **お知らせの表示**: `resident-statement.json` の固定 `notice` を `NoticeBlock` が描画している
- **印刷フロー**: 利用者詳細 → 印刷ボタン → `/print/preview?residentId=&year=&month=&type=resident` → `resident-statement` API → `PdfRenderer` で `template.notice` をそのまま使用
- **施設データ**: 印刷APIで `resident.facility` を取得しており、施設単位の設定を参照可能
- **施設マスタ**: フォームと API の拡張パターンが既存実装と整合している

---

## 2. 現状の整理

| 項目 | 現状 |
|------|------|
| お知らせの定義 | `pdf/templates/resident-statement.json` に固定で3行 |
| 利用者詳細の印刷 | 「🖨️ 印刷」ボタンのみ |
| 印刷プレビューURL | `type=resident` のみ（通常/退居の区別なし） |
| 施設テーブル | `noticeTemplateNormal`, `noticeTemplateMoveOut` なし |

---

## 3. 実装計画

### Phase 1: DB スキーマの更新

**対象ファイル**: `prisma/schema.prisma`

**追加カラム**:
- `noticeTemplateNormal` (String?, 通常用お知らせテンプレート)
- `noticeTemplateMoveOut` (String?, 退居用お知らせテンプレート)

**データ形式**: 改行区切りのテキスト。表示時に `\n` で分割して `lines` 配列として扱う。

**マイグレーション**: 新規マイグレーションで上記2カラムを追加（既存レコードは `null`）。

---

### Phase 2: 施設マスタのフォーム拡張

**対象ファイル**: `app/master/page.tsx`

**追加項目**:
1. 【通常】預り金明細書のお知らせテンプレート（textarea）
2. 【退居】預り金明細書のお知らせテンプレート（textarea）

**配置**: 役職者の名前の下、表示の順番の上あたり。

**仕様**:
- 複数行入力（改行で1行ずつ表示）
- 空欄可（空の場合はデフォルト文言を使用）
- 最大文字数: 500文字程度（要検討）

**フォーム状態**: `facilityForm` に `noticeTemplateNormal`, `noticeTemplateMoveOut` を追加。

---

### Phase 3: 施設 API の更新

**対象ファイル**:
- `app/api/facilities/route.ts` (POST)
- `app/api/facilities/[id]/route.ts` (PUT)

**対応内容**:
- `noticeTemplateNormal`, `noticeTemplateMoveOut` の受け取り・保存
- バリデーション（最大長など）
- `lib/validation.ts` に `NOTICE_TEMPLATE` の最大長を追加

---

### Phase 4: 印刷 API の更新

**対象ファイル**: `app/api/print/resident-statement/route.ts`

**対応内容**:
- クエリパラメータ `noticeType` を追加（`normal` | `moveout`、未指定時は `normal`）
- 施設の `noticeTemplateNormal` / `noticeTemplateMoveOut` を参照
- レスポンスの `ResidentPrintData` に `notice` を追加

**データ構造**:
```typescript
// ResidentPrintData に追加
notice?: {
  title: string      // "【お知らせ】"
  lines: string[]    // 改行で分割した行
}
```

**ロジック**:
- `noticeType === "moveout"` → `noticeTemplateMoveOut`
- それ以外 → `noticeTemplateNormal`
- 施設のテンプレートが空/null の場合は、`resident-statement.json` のデフォルト文言を使用

---

### Phase 5: transform の更新

**対象ファイル**: `pdf/utils/transform.ts`

**対応内容**:
- `ResidentPrintData` に `notice?: { title: string; lines: string[] }` を追加
- `transformToResidentPrintData` の引数に `noticeType` と施設のテンプレートを渡す
- テンプレート文字列を改行で分割して `notice` を組み立てる

※ テンプレートの解決は API 側で行い、`transform` には解決済みの `notice` を渡す形でも可。どちらがシンプルかで決定。

---

### Phase 6: 利用者詳細画面の UI 更新

**対象ファイル**: `app/residents/[id]/page.tsx`

**対応内容**:
- 「🖨️ 印刷」ボタン（既存）: `/print/preview?...&type=resident` のまま（通常）
- 「退居向け印刷」ボタン（新規）: `/print/preview?...&type=resident&noticeType=moveout`

**レイアウト**: 印刷ボタンの横に「退居向け印刷」を追加。

---

### Phase 7: 印刷プレビュー画面の更新

**対象ファイル**: `app/print/preview/page.tsx`

**対応内容**:
- `noticeType` を URL パラメータから取得
- `fetchResidentPrintData` の URL に `&noticeType=normal` または `&noticeType=moveout` を付与

---

### Phase 8: PDF レンダラーの更新

**対象ファイル**:
- `pdf/renderer/PdfRenderer.tsx`
- `pdf/renderer/BatchPdfRenderer.tsx`

**対応内容**:
- `resident-statement` の場合、`data.notice` があればそれを使用
- `data.notice` がなければ `template.notice`（デフォルト）を使用

```typescript
// 変更前
<NoticeBlock notice={template.notice} />

// 変更後
<NoticeBlock notice={data.notice ?? template.notice} />
```

---

### Phase 9: まとめて印刷の対応

**対象ファイル**: `app/api/print/batch-print/route.ts`

**対応内容**:
- まとめて印刷は「通常」のみとし、`noticeType` は常に `normal`
- 各利用者の `ResidentPrintData` に、施設の `noticeTemplateNormal` から生成した `notice` を含める

---

## 4. 影響範囲の整理

| 対象 | 変更内容 |
|------|----------|
| DB | Facility に 2 カラム追加 |

| 利用者詳細 | 退居向け印刷ボタン追加 |

| BatchPdfRenderer | 同上 |

---

## 5. デフォルト文言の扱い

施設のテンプレートが空または未設定の場合、現在の固定文言を使用する。

```
いつもお世話になっております。
先月の【預り金明細書】を送付させて頂きます。
尚、残高が10,000円以下となられた方は、早急に補充の程よろしくお願い致します。
```

この文言は `resident-statement.json` に残し、`data.notice` が無い場合のフォールバックとして利用する。

---

## 6. バリデーション・制約

- お知らせテンプレート: 最大 500 文字（要検討）
- `noticeType`: `normal` または `moveout` のみ許可

## 8. 確認事項（実装前に決めたい点）

1. **お知らせの最大文字数**: 500 文字で問題ないか
2. **退居向けのデフォルト**: 施設が退居用テンプレートを未設定の場合、通常用を使うか、別のデフォルト文言を使うか
3. **まとめて印刷**: 退居向けのまとめて印刷は必要か（現計画では通常のみ）
