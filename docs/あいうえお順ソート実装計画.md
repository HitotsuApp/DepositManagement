# あいうえお順ソート 実装計画

## 概要

利用者の並び替えに「あいうえお順」を追加する。正確なソートにはふりがな（読み仮名）が必要なため、Resident にオプション項目として `nameFurigana` を追加する。

**ふりがな自動入力は実装しない**（遅延・精度のリスク、入力頻度が低いため）。

---

## 前提条件

| 項目 | 内容 |
|------|------|
| ふりがな | 必須ではない（オプション） |
| ふりがな未設定時 | 利用者名（name）でソート（localeCompare でフォールバック） |
| 対象 | 利用者（Resident）のみ（ユニットは今回対象外） |

---

## ふりがな入力の制御

### 許容する文字

| 種類 | 文字 | 用途 |
|------|------|------|
| ひらがな | ぁ〜ん、ゔ、ゕ、ゖ | 基本（ゔは外来語用） |
| 中黒 | ・ | ミドルネーム（例: まりー・くりすてぃーぬ） |
| 長音 | ー | 長音（例: おおさか） |

### 禁止する文字

- **スペース**（半角・全角）: 禁止
- **カタカナ**: 入力時にひらがなへ自動変換（混在禁止）

### 入力欄の制御

- **入力時（onChange）**: カタカナをひらがなに変換
- **許容チェック**: ひらがな・中黒・長音以外は入力しない（または入力時に除去）
- **表示**: 「ひらがな、ー、・のみ入力可」と表記し、利用者に理解してもらう

### 保存時の処理

- API 側でもカタカナ→ひらがな変換を実施（インポート等の経路対策）
- DB にはひらがな・中黒・長音のみが保存される

## Phase 1: DB スキーマ変更

### 1.1 Resident に nameFurigana を追加

**ファイル**: `prisma/schema.prisma`

```prisma
model Resident {
  // 既存フィールド...
  name                  String        // 最大30文字
  nameFurigana          String?       // ふりがな（最大50文字、ひらがな・ー・・のみ、オプション）
  displaySortOrder      Int?
  printSortOrder        Int?
  // ...
}
```

- **nameFurigana**: ひらがな・中黒・長音のみ（DB 保存前に正規化）
- **最大文字数**: 50文字
- **NULL 許容**: オプションのため必須ではない

### 1.2 Facility に residentSortMode を追加

```prisma
model Facility {
  // 既存...
  residentSortMode      String?       // 'manual' | 'aiueo' | null(manual扱い)
}
```

### 1.3 マイグレーション

```bash
npx prisma migrate dev --name add_aiueo_sort
```

## Phase 3: ソートロジックの拡張

### 3.1 lib/sortOrder.ts の拡張

**新規関数**: `sortResidentsByAiueo`

- nameFurigana を優先、未設定時は name でソート
- `localeCompare(str, 'ja')` で五十音順
- DB 保存時点でひらがなに統一されているため、ソート時の正規化は不要

**既存関数の拡張**: `sortResidentsForPrint` 等に residentSortMode を渡し、aiueo の場合は `sortResidentsByAiueo` を呼ぶ分岐を追加。

## Phase 4: API 変更

### 4.1 利用者 API

**ファイル**: `app/api/residents/route.ts`, `app/api/residents/[id]/route.ts`

- GET: `nameFurigana` をレスポンスに含める
- POST/PUT: `nameFurigana` を受け付け、保存前に `sanitizeFurigana` + `katakanaToHiragana` で正規化

### 4.2 施設 API

**ファイル**: `app/api/facilities/route.ts`, `app/api/facilities/[id]/route.ts`

- GET: `residentSortMode` をレスポンスに含める
- PUT: `residentSortMode` の更新を受け付け

### 4.3 施設詳細 API

**ファイル**: `app/api/facilities/[id]/route.ts`

- residents 取得時に `nameFurigana` を include
- 施設の `residentSortMode` に応じてソートロジックを切り替え

### 4.4 印刷 API

**ファイル**: `app/api/print/batch-print/route.ts`, `app/api/print/deposit-statement/route.ts`

- 施設の `residentSortMode` を取得
- aiueo の場合は `sortResidentsByAiueo` を使用

---

## ソート時の優先順位（aiueo モード時）

1. ユニット順（useUnitOrderForPrint が true の場合）
2. あいうえお順: nameFurigana ?? name で `localeCompare(str, 'ja')`
3. 同順位時: id で昇順

---

## ふりがな未設定時の扱い

- ふりがな未設定: name でフォールバック（`localeCompare(name, 'ja')`）
- 漢字を含む name のソートは読みが不正確になるが、未設定時は許容

## 検討事項

1. **ユニットのあいうえお順**: 今回の要望は利用者のみ。
2. **既存データ**: nameFurigana は NULL のまま。residentSortMode は null または 'manual' で従来通り。
3. **ふりがな自動入力**: 遅延・精度のリスク、入力頻度が低いため**実装しない**。
