# 預り金システム：エクセル取込仕様書

このドキュメントは、施設で使用されている預り金出納帳（Excel）を、外部システムへインポートするための技術仕様書です。

## 1. 対象ドキュメント
- **ファイル名**: `2025年度(R7)   笑生苑預り金出納帳.xls`
- **対象シート**: `4月分　金銭出納帳` （または見本シート）
- **シート形式**: 固定レイアウト（261行 × 15列）

## 2. 列定義 (Column Mapping)

| 列 (Index) | 項目名 | データ型 | 内容・備考 |
| :--- | :--- | :--- | :--- |
| **A (0)** | ユニット名 | 文字列 | ユニット名（例：右田、佐野） |
| **B (1)** | 利用者名 | 文字列 | 利用者のフルネーム |
| **C (2)** | 月 | 数値 | 取引日の「月」（例：4） |
| **D (3)** | (月ラベル) | 文字列 | 「月」または「月分」という固定文字 |
| **E (4)** | 日 | 数値 | 取引日の「日」 |
| **F (5)** | (日ラベル) | 文字列 | 「日」という固定文字 |
| **G (6)** | 種別 | 文字列 | 取引カテゴリ（例：医療、買物、入金、外出、合計） |
| **H (7)** | 摘要 | 文字列 | 取引の具体的な内容 |
| **I (8)** | 支払先 | 文字列 | 購入店や医療機関名 |
| **J (9)** | 入金 | 数値 | 入金額（空の場合はNaN/0） |
| **K (10)** | 出金 | 数値 | 出金額（空の場合はNaN/0） |
| **L (11)** | 残高 | 数値 | 取引後の残高（システム計算値との照合用） |

## 3. 行の構造と判別ロジック

### 3.1 ヘッダー行 (Row 1 / Index 1)
- **特徴**: 列名が記載されている。
- **処理**: インポート時はこの行を読み飛ばすか、バリデーションに使用する。

### 3.2 繰越行 (Row 2 / Index 2)
- **特徴**: 摘要（H列）に「前年度より繰越」または「前月より繰越」と記載。
- **重要セル**: L列に初期残高が入る。

### 3.3 データ入力行 (Rows 3-260 / Index 3-259)
- **有効なデータの判別**:
  - `摘要(H列)` または `種別(G列)` に値が入っている行を有効なデータとみなす。
  - C列（月）とE列（日）を結合して日付を作成する。
- **数値処理**:
  - J列（入金）、K列（出金）は空文字やNaNを0として扱う。

### 3.4 合計行 (Row 261 / Index 260)
- **特徴**: G列に「合計」という文字が含まれる。
- **バリデーション用セル**:
  - **J261**: 取込データの入金合計と一致すべき値。
  - **K261**: 取込データの出金合計と一致すべき値。
  - **L261**: 最終的な残高。

## 4. 実装における注意点
1. **日付の合成**: プログラム側で `Date(Year, C列, E列)` のように組み立てる必要がある（年度はファイル名や別セルから取得）。
2. **シート名の特定**: ユーザが「4月分」を選択した場合、実際のシート名は全角・半角の混在や末尾スペースを考慮し、`sheet_name.startswith("4月分")` などで柔軟に特定すること。
3. **空行**: 入力行の多くは空（NaN）であるため、フィルタリングが必須。
